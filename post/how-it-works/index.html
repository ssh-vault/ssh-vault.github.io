How it works


ssh-vault will randomly create a password and use it for encrypting the contents of the vault using a symmetric cipher AES256, the password later will be encrypted using the ssh-rsa public key of the receiver.

This simplify the share of sensitive data across developers, teams, or either to store things on a public repository by just using ssh keys.

The option -u will fetch the receiver ssh-rsa public from github:

 https://github.com/<username>.keys

This helps to create a vault very straight forward, since the both the sender and receiver don’t need to exchange their keys before in order to share data, therefor this helps to work asynchronously, the only requirement is that the receiver needs to have an ssh-rsa public key/account on github.com

Creating a vault (encryption)

Example:

User Alice wants to send something to user Bob

Alice will type:

$ ssh-vault -u bob create vault

By doing this Alice will fetch for bob ssh public keys stored in github https://github.com/bob.keys and create a vault that only bob will be available to decrypt by using his private ssh key

In case Alice want’s to use a different key, could do:

$ ssh-vault -k /path/to/bob/id_rsa.pub create vault

Edit/View a vault (decryption)

Continuing with the previous example, if Bob want to see what Alice sent to him, just need to type:

$ ssh-vault view vault

He will be prompted for his ssh-key private password, or if he wants to use another private key, he can type:

$ ssh-vault -k ~/.ssh/id_rsa_other view vault
